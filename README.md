# c-frs

Clojure application for generating tower running / stair climb ranking sheets from race result files.

The system reads race data from `TowerRunningRaceData/`, normalizes athlete identities, computes points-based rankings, and writes HTML result sheets into `content/`.

Main entry point: `c-frs.core/-main` in `src/c_frs/core.clj`

---

## Overview

This project:

- scans `TowerRunningRaceData/` for race files (`.csv` and `.json`)
- filters races to a recent scoring window
- normalizes athlete names via `TowerRunningRaceData/translate.dat`
- marks foreign athletes via `TowerRunningRaceData/foreign.dat`
- computes rankings for:
  - `:male` and `:female`
  - all countries and US-only
  - all ages and standard age buckets
- writes HTML pages to `content/`

Examples of generated output:

- `content/male-all-all.html`
- `content/male-all-us-only.html`
- `content/female-40-49-all.html`
- `content/races-considered.html`

- `TowerRunningRaceData/` — race source files and supporting lookup files
- `TowerRunningRaceData/translate.dat` — name translation / normalization rules
- `TowerRunningRaceData/foreign.dat` — list of foreign athlete names

- `content/` — generated HTML files

### Helper scripts

- `parse-runsignup.py` — converts RunSignup JSON exports into the JSON format consumed by this app
- `scoring` — runs the built standalone JAR
- `sync` — rsyncs generated site content to a remote server

## High-level pipeline

The main flow in `src/c_frs/core.clj` is:

1. `scan-directories`
    - finds `.csv` and `.json` race files under `TowerRunningRaceData/`

2. `read-race`
    - dispatches to:
        - `read-csv-race`
        - `read-json-race`

3. athlete parsing / normalization
    - `athlete-from-row`
    - `normalize-athlete`
    - `translate-name`
    - `foreign-name?`

4. race-level cleanup
    - `dedupe-athletes`
    - `dedupe-separate`

5. scoring
    - `add-scores-and-rank`
    - `compute-sex-results`
    - `compute-overall-result-sheet-from-races`

6. athlete aggregation
    - `partition-athlete`
    - `deduplicate-by-race-max-points`
    - `create-athlete-row`

7. sheet generation
    - `filter-ages`
    - `add-row-ranks`
    - `print-to-file`

8. orchestration
    - `make-sheets`
    - `-main`

## Scoring model

### Race points

Each race has a base point value from its header, for example `150`, `200`, or `250`.

The scoring fractions are generated by `fractions`:

- 1st place: `5/5`
- 2nd place: `5/6`
- 3rd place: `5/7`
- 4th place: `5/8`
- etc.

`get-scores-list` multiplies those fractions by the race’s base point value.

So in a 150-point race:

- 1st = `150`
- 2nd = `125`
- 3rd = `750/7`
- etc.

### US-only scoring

US-only sheets are not created by simply filtering foreigners at the end.

Instead, in `compute-sex-results`, foreign athletes are removed before race scoring is assigned. That means Americans are re-ranked within the reduced field, and points are recomputed accordingly.

Example:

- if a foreign athlete wins overall
- then the top American is promoted to 1st in the US-only sheet
- and gets full first-place points for that version

This is an intentional part of the design.

### Athlete totals

For each athlete:

- race results are deduplicated by race name using `deduplicate-by-race-max-points`
- the top 5 scoring races are summed in `create-athlete-row`

## Identity handling

### Name normalization

Name normalization is done by `translate-name`, built from `name-translator-factory`.

Rules come from `TowerRunningRaceData/translate.dat`.

These are regex-based mappings used to merge alternate spellings / variants into a canonical athlete name.

### Foreign athletes

Foreign athlete detection is done by `foreign-name?`, built from `foreign-marker-factory`.

Names are checked against `TowerRunningRaceData/foreign.dat`.

### Same-name athletes with different ages

If multiple race entries share a name but appear to represent different people, `partition-athlete` tries to separate them using age compatibility rules.

Current logic in `ages-compatible?` treats athletes as compatible if:

- either age is missing, or
- ages differ by at most 1 year

This is a pragmatic heuristic for rare collisions.

## Input formats

## CSV races

CSV races are read by `read-csv-race`, which eventually calls:

- `read-csv-file-into`
- `get-race-from-csv-strings`
- `athlete-from-row`

Input rows are cleaned with `clean-line`, which currently trims and uppercases cell values.

## JSON races

JSON races are read by `read-json-race`.

Expected structure:

1. first JSON value: header object
2. second JSON value: array of athletes

The header is read separately from the athlete list.

Relevant transforms are applied in `value-fn`, including:

- `:date`
- `:name`
- `:sex`
- `:gender`

---

## Output format

Generated pages are written to `content/` by `print-to-file`.

Filenames follow this pattern:

```text
<sex>-<age-range>-<all|us-only>.html
```

Examples:

- `male-all-all.html`
- `male-40-49-us-only.html`
- `female-20-29-all.html`

There is also:

- `races-considered.html`

which lists the races included in the current scoring window.

---

## Age groups

Defined in `age-ranges` in `src/c_frs/core.clj`:

- `0-19`
- `20-29`
- `30-39`
- `40-49`
- `50-59`
- `60-69`
- `70-79`
- `80-89`
- `90-99`
- `100-200`

Age filtering is done by `filter-ages`.

---

## Ranking display

After totals are computed and sorted, `add-row-ranks` assigns displayed ranks.

It handles ties in “competition ranking” style:

- `1, 2, 2, 2, 5, 5, 7, ...`

It also alternates background colors by tie group for readability in the generated HTML.

---

## Recent-race filter

The scoring window is controlled by:

- `one-year-ago`
- `recent-enough?`

At runtime, a race is included only if its date is after the computed cutoff.

If output seems incomplete, this is one of the first places to check.

---

## RunSignup conversion helper

`parse-runsignup.py` converts a RunSignup results export into the JSON format read by `read-json-race`.

Usage:

```bash
python3 parse-runsignup.py input-results.json output-race.json
```

Typical workflow:

1. export or download race results JSON
2. convert with `parse-runsignup.py`
3. place the output file into `TowerRunningRaceData/`
4. rerun the scorer with `lein run`

## Troubleshooting

### `TowerRunningRaceData/` is missing

Initialize the submodule:

```bash
git submodule update --init --recursive
```

### Submodule fetch fails

Switch to HTTPS:

```bash
git submodule set-url TowerRunningRaceData https://github.com/davidhanley/TowerRunningRaceData.git
git submodule update --init --recursive
```

### `lein run` produces fewer races than expected

Check:

- `recent-enough?`
- race dates in the source files
- whether the source files are valid CSV/JSON

### Athlete names are not collapsing as expected

Check `TowerRunningRaceData/translate.dat`.

### Foreign athletes are not being marked correctly

Check `TowerRunningRaceData/foreign.dat`.

### US-only rankings look lower than expected

Remember that US-only sheets are re-scored after foreign athletes are removed, rather than just filtered from the final output.
